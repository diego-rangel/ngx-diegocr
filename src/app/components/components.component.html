Input Select

Never more get confused when using a number as your ngModel.

The InputSelect component supports not only a template mode, but an options input too. You can use an array of objects of any types, you just need to set the displayTextProperty and valueProperty values.

Input Date

It automatically handles string dates into the model.

In order to localize the BsDatePicker from ngx-bootstrap, you must import your locale as described into the Get Started section.

Inputs

Name Type Description
placeholder string Input placeholder.
minDate Date Minimum date allowed.
maxDate Date Maximum date allowed.
format string The date format to show.
theme
Default: theme-dark-blue 		string Defines the layout colors of the picker.

Outputs

Name Description
onChange Emitted every time a date is changed with the new value as a parameter.
Input Upload

Inputs

Name Type Description
url
Required 		string The URL where the selected file should be uploaded.
autoUpload
Default: true 		boolean Defines when the component should automatically POST the selected file or not.
label string The input's label.
disabled
Default: false 		boolean Disables the component.
placeholder
Default: Select a file to upload... 		string The input's placeholder.
dropZonePlaceholder
Default: Drag & drop a file to import. 		string The Dropzone's placeholder.
showDropZone
Default: false 		boolean Defines if the Dropzone should be visible or not.
showQueue
Default: false 		boolean Defines if the Files queue should be visible or not. Currently only visible when chunking is enabled.
chunk
Default: false 		boolean Defines when the selected file should be chunked or not. If true, the selected file will be splitted into N parts according to a chunkSize.
chunkRetries
Default: 3 		number Defines how many times each chunk part should be retried in case of errors.
chunkSize
Default: 1048576 		number Defines the max size for each chunk part in bytes.
chunkRequestsCountInParallel
Default: 50 		number Defines the max number of chunks to send in parallel.
maxFileSize
Default: 0 		number Defines the max file size limit in megabytes. Omit or use 0 for no limit.
withCredentials
Default: false 		boolean Defines if the POST requests should contain the withCredentials parameter.
selectButtonIcon
Default: fa fa-folder 		string The select button's icon.
selectButtonLabel
Default: Select 		string The select button's label.
removeButtonIcon
Default: fa fa-trash 		string The remove's button icon.
removeButtonLabel
Default: Remove 		string The remove's button label.
fileTypeErrorMessage
Default: The file type [{{'{extension}'}}] is not allowed. 		string The error message to show when the file type is invalid.
fileSizeErrorMessage
Default: This file exceeds the max file size allowed of {{'{maxFileSize}'}}MB. 		string The error message to show when the file size is invalid.
maxFileSizeLabel
Default: Max file size: 		string The max file size label.
allowedExtensionsLabel
Default: Allowed extensions: 		string The allowed extensions label.
allowedExtensions string[] The allowed extensions.

Outputs

Name Description
onFileAdded Emitted every time a file is selected.
onUploadComplete Emitted when the selected file is completely uploaded. When chunking, it's called when all parts were successfully sent.
onChunkFileUpload Emitted when chunk is true, for each successfully chunk part upload.
onError Emitted every time an error occurs.
onClear Emitted every time the component is cleared.
